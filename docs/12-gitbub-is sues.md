# 12 gitbub is sues（新一波任务规划）

> 说明：按你的要求，聚焦 **Client 端功能可用化** 的下一波任务，不再停留在页面框架。

## Wave 2 任务清单

### W2-P0-01 客户登录体验强化
- 增加图形验证码/风控阈值（按后端能力择一）。
- 增加登录审计日志与异常登录提示。
- 补充“记住我”与多端登录策略说明。

### W2-P0-02 题目生成质量控制
- 增加输入约束模板（学校、专业、字数、研究方向）。
- 增加“再生成但保留偏好”能力。
- 增加结果评分与人工改写建议入口。

### W2-P0-03 开题报告结构化输出
- 固化章节模板（研究背景、目的意义、方法、计划）。
- 输出结构化 JSON + 可渲染文档，便于前端多视图展示。
- 增加“导师反馈后再次生成”对比能力。

### W2-P0-04 正文生成长任务稳定性
- 增加任务超时、取消、断点续跑策略。
- 增加阶段产物缓存，避免全量重跑。
- 增加失败自动诊断（参数不足、敏感词、模型超时等）。

### W2-P0-05 交付物验收与下载安全
- 增加下载水印与下载记录。
- 增加文件完整性校验（hash）。
- 增加下载链接短时效与二次校验。

### W2-P1-01 客户工作台可观测性
- 展示最近任务耗时、成功率、失败类型分布。
- 管理端可按用户/渠道查看生成链路健康度。

---

## 建议执行顺序
1. W2-P0-01（登录稳定）
2. W2-P0-02（题目质量）
3. W2-P0-03（开题结构）
4. W2-P0-04（正文稳定）
5. W2-P0-05（交付与下载）
6. W2-P1-01（可观测性）

## 每个任务 PR 最低要求
- 明确“机构视角、学生视角、管理视角”三方影响。
- 提供本地验证步骤（Docker、web/api lint/test/build、关键页面路径）。
- 提供回滚方案与风险点。
