# 03 功能架构（Feature Architecture）

| 模块 | 优先级 | 当前状态 | 后续动作 |
|---|---|---|---|
| 认证与权限 | P0 | 后端较完整，前端后台已接入 | 增加机构角色与机构数据权限边界 |
| 学生端下单与工作台 | P0 | 能力存在但前端不完整 | 补全 C 端闭环页面 |
| 机构端推单/代下单 | P0 | 需求明确，产品层缺失 | 新增机构端入口、线索与代下单流程 |
| 商品与分类 | P0 | 管理端可维护 | 同步接入 C/B 端展示和购买引导 |
| 订单与支付 | P0 | 后端有订单+支付+退款 | 增加机构下单场景与支付主体规则 |
| 任务状态机 | P0 | 状态定义完整，管理端可操作 | 增强状态迁移审计、补偿机制 |
| 题目/开题/大纲/写作 | P1 | 后端能力已存在 | 增加阶段交付与返工协作可视化 |
| 参考文献 | P1 | 后端模块存在 | 增加前端引用校验与编辑体验 |
| 导出模块 | P1 | 后端 DOCX 能力存在 | 增加用户下载中心与机构可见性策略 |
| Prompt 资产管理 | P1 | 管理后台已有 | 增加版本审批与灰度发布 |
| 配额系统 | P1 | 后端有 quota | 增加机构渠道配额/额度策略（需确认） |
| 仪表盘与报表 | P2 | 管理端已有基础 | 增加渠道转化与机构绩效分析 |

## 说明
- 当前“后台能力 > 前台产品能力”。
- P0 主目标：形成“学生/机构下单 → 执行交付 → 修改完成”的双通道闭环。
