# 07 UI 设计规范（UI Guidelines）

## 1. 整体视觉风格
- 简洁专业、信息密度中高。
- 管理后台优先可读性与操作效率。
- 机构端与学生端保持同一品牌体系，但信息层级可区分。

## 2. 页面布局规则
- 学生端：任务进度与交付结果优先。
- 机构端：线索、订单进度、协同备注优先。
- 管理端：筛选、批量操作、审计信息优先。
- 列表页：筛选区 + 表格区 + 分页区。
- 详情页：摘要信息 + 时间线 + 操作区。

## 3. 表单规范
- 必填项明确标记。
- 高危操作（状态覆盖、退款）必须二次确认。
- 机构代下单需校验学生基本信息完整性。
- 表单提交后给出 loading 与结果反馈。

## 4. 表格规范
- 固定最小列集合（编号、来源、状态、更新时间、操作）。
- 支持批量操作前必须有选择反馈与风险提示。
- 渠道来源字段必须可筛选（direct/agency）。

## 5. 按钮规范
- 主按钮：主流程动作（保存、提交、下单）。
- 次按钮：辅助动作（取消、重置）。
- 危险按钮：删除、强制状态变更（红色 + 确认弹窗）。

## 6. 状态规范
- loading：骨架屏优先。
- empty：展示空态文案与引导动作。
- error：错误信息 + 重试。
- success：toast + 关键字段回显。

## 7. 移动端适配规范
- 管理后台：最小支持平板横屏。
- 学生端与机构端：必须支持手机端断点（>=375px）。
- 表格在移动端改为卡片列表展示。
