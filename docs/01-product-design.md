# 01 产品设计（Product Design）

## 1. 产品定位
“论文任务生产平台”：
- 为**学生用户（To C）**提供论文任务生成与服务交付流程；
- 为**合作机构（To B）**提供推单、代下单、进度协同、交付跟踪能力；
- 为**内部运营/管理员**提供订单、任务、用户、内容资产管理能力。

## 2. 目标用户
1. 学生用户（下单与任务消费者）
2. 合作机构人员（渠道推单与代下单）
3. 导师/写手（任务执行者）
4. 运营与管理员（质检、分配、售后、配置）

## 3. 核心场景
- 学生直接注册登录后购买商品并支付。
- 机构人员代学生创建订单或推送线索转订单。
- 系统创建论文任务并推进状态（题目→开题→大纲→写作→交付）。
- 管理后台进行任务分配、状态纠偏、退款处理、配额管理。
- 按不同导师要求多轮调整，最终导出与交付。

## 4. 核心业务流程
1. 注册/登录（学生/机构）
2. 浏览服务包并下单（自助下单或机构代下单）
3. 支付与订单确认
4. 自动/人工创建并关联任务
5. 分配写手/导师
6. 题目交付 → 开题报告交付 → 正文交付
7. 按导师要求多轮修改
8. 用户预览/下载导出结果
9. 售后（退款、重试、纠偏）

## 5. 页面结构（目标态）
### C 端（学生）
- 首页/营销页
- 登录注册
- 商品列表与详情
- 订单页
- 任务工作台（状态时间线 + 阶段结果）
- 导出下载中心
- 账户中心

### B 端（机构）
- 机构工作台（转化漏斗、在制订单）
- 代下单入口
- 推单线索管理
- 订单/任务协同页
- 交付进度追踪
- 结算与对账页（后续）

### 内部管理后台（当前主实现）
- 仪表盘
- 订单管理
- 任务管理
- 用户管理
- 商品/分类/门店管理
- Prompt 资产管理

## 6. MVP 范围
- 登录注册（邮箱/手机号至少一种）
- 学生自助下单 + 机构代下单
- 商品下单支付（至少 1 种支付通道可用）
- 订单-任务关联
- 任务状态可视化推进
- 题目/开题/正文阶段性交付
- 导出 DOCX
- 管理后台核心运营动作

## 7. 非 MVP 范围
- 多租户 SaaS 化
- 高级 BI 报表
- 全渠道消息中心
- 复杂客服工单体系
- 机构自动分润结算系统（可 P1/P2）
