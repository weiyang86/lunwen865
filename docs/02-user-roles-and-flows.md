# 02 用户角色与流程（User Roles & Flows）

## 1. 角色定义
- USER：普通学生用户
- VIP：高级学生用户
- AGENCY_STAFF：合作机构人员（推单/代下单）
- TUTOR：导师/写手
- ADMIN：管理员
- SUPER_ADMIN：超级管理员

## 2. 权限边界（建议）
- USER/VIP：查看与管理自己的订单、任务与导出。
- AGENCY_STAFF：代下单、查看所属机构订单/任务进度、提交协同备注。
- TUTOR：查看被分配任务，提交阶段产出（题目、开题、正文、修改稿）。
- ADMIN：管理订单、任务、用户、商品、退款、任务分配。
- SUPER_ADMIN：具备全局配置、权限管理与高危操作权限。

## 3. 核心流程
### 3.1 学生自助下单流程（To C）
注册/登录 → 选择商品 → 提交订单 → 支付成功 → 创建/绑定任务 → 分阶段交付 → 修改 → 完成。

### 3.2 机构推单/代下单流程（To B）
机构登录 → 创建线索或代下单 → 支付确认（机构或学生）→ 进入任务池 → 分配写手/导师 → 跟踪交付进度 → 完成。

### 3.3 任务生产流程（统一）
INIT → TOPIC_* → OPENING_* → OUTLINE_* → WRITING → MERGING/FORMATTING → REVIEW/REVISION → DONE。

### 3.4 售后流程
退款申请 → 审核（同意/拒绝）→ 渠道退款结果回调 → 状态同步。

## 4. 管理操作流程
- 任务分配：管理员指定 assignee（写手/导师）。
- 状态纠偏：管理员带理由修改任务状态。
- 订单关联：将历史订单绑定到任务。
- 机构协同：管理员可查看机构侧备注与交付反馈。

## 5. 需要人工确认
- AGENCY_STAFF 是否可查看完整正文，还是仅查看里程碑状态。
- TUTOR 是否可以直接修改任务状态，还是仅提交产出待审核。
- 机构代下单时支付主体与发票主体规则。
