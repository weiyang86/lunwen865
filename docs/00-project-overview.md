# 00 项目总览（Project Overview）

## 1. 项目定位
本项目是一个“论文生产与交付平台”，同时服务两类业务：
- **To C（学生用户）**：学生直接下单并跟踪论文任务交付。
- **To B（机构合作）**：专升本/教育机构作为合作渠道，进行推单或代下单，并跟进执行进度。

平台覆盖从下单支付、论文任务推进、AI 生成（题目/开题/大纲/写作/润色）、参考文献、导出，到后台运营管理的完整业务链路。

## 2. 当前业务模式（本阶段）
1. 学生用户下单。
2. 管理后台分配给写手/导师。
3. 分阶段交付题目、开题报告、论文正文。
4. 按不同导师要求进行修改与返工。
5. 合作机构可推单/代下单后进入同一执行交付流程。

## 3. 代码现状摘要（基于仓库）
- 前端：`apps/web`，Next.js 16 + React 19，当前以管理后台为主。
- 后端：`apps/api`，NestJS 11，已拆分多个业务模块。
- 数据库：PostgreSQL + Prisma。
- 队列：BullMQ + Redis（用于异步任务）。

> 注：C 端与机构端前台页面在本仓库中尚未形成完整产品闭环，首页仍为基础联通示例。

## 4. 当前阶段判断
- **能力层**：后端能力较完整（任务状态机、订单支付、管理模块）。
- **产品层**：管理后台可用，C 端/机构端产品层不足。
- **上线层**：缺统一部署/可观测性/验收闭环文档与演练。
